Host discovery con Ping ARP (Address resolution protocol)
    Solo se posible se se encuentra en la misma red
    ARP pide las MAC address de los equipos que hay en la red
        # nmap -n -sn -PR <IPs>




Host discovery con Ping ICMP (internet Control Message protocol)
    Antes de realizar un ICMP tiene que ejecutarse una resolución ARP si está en la misma red    
    Envia un ICMP Echo Request, filtrado por los firewall. Espera recibir un ICMP echo Reply  
        # nmap -n -sn -PE <IPs>
    Envia un ICMP Timestamp Request, menos filtrado que el anterior, espera recibir un ICMP Timestamp Reply
        # nmap -n -sn -PP <IPs>
    Envia un ICMP Address Mask Request, hay cortafuegos no lo filtran, espera recibir un ICMP AddMask reply
        # nmap -n -sn -PM <IPs>



Host discovery con Ping TCP SYN
    EN este tipo de comandos para obtener la maxima funcionalidad es necesario lanzarlos como usuario
    privilegiado, si no es así usa un TCP SYN Connect()
    Utiliza la comunicación Tree Way Handshake, 
    Enviando un paquete TCP vacío con la bandera SYN (sincronizar) al puerto 80 aunque es configurable
        La respuesta puede ser SYN/ACK (Acknowledge) o RST
    Se utliza el comando
        # nmap -n -sn -PS <IPs>
        # nmap -n -sn -PS21,22,23,25,80,443 <IPs>



Host discovery con Ping TCP ACK
    EN este tipo de comandos para obtener la maxima funcionalidad es necesario lanzarlos como usuario
    privilegiado, si no tiene permisos ejecuta un TCP SN connect()
    Utiliza la comunicación Tree Way Handshake, pero indicando que ya hay una comunicación iniciada, para
    saltarse los firewalls 
    Enviando un paquete TCP vacío con la bandera ACK (Acknowledge) al puerto 80 aunque es configurable
        La respuesta puede ser RST si el equipo está encendido
    Se utliza el comando
        # nmap -n -sn -PA <IPs>
        # nmap -n -sn -PA21,22,23,25,80,443 <IPs>
    Para saltarse un firewall se suele enviar un PING SYN y ACK a la vez



Host discovery con Ping UDP
    Se envia un paquete UDP a los puertos que se indiquen en el comando, si no indicamos puertos se envian
    al puerto 31338.
    Si el puerto está cerrado llega una respuesta de puerto no alcanzables está abierto no se espera
    respuesta.
    Hay puertos que solo filtran TCP, por lo que usar UDP es un metodo para saltarlos
    El comando a ejecutar seria:
        # nmap -n -sn -PU <IPs>
        # nmap -n -sn -PU21,22,23,24,25,


El comando para ejecutar
    # nmap -Pn -n -sS <IPs>
El comando para ejecutar sobre un puerto determinado
    # nmap -PN -n -sS p xxx <IPs>


Para la detección de sistema operativo se utiliza el siguiente comando:
    # nmap -T4 -O <IPs>
    # nmap -T4 -O -Pn <IPs>
    #nmap -T4 --script smb-os-discovery.nse 10.10.1.22
    intenta determinar OS, nombre de equipo, grupo de trabajo sobre SMB (445, 139)
        --osscan-guess --osscan-limit o --fuzzy para aproximar en caso de no encontrar los S.O


Si queresmo hacer detección de servicios, S.O, traceroute y scripts de escaneo de información en un comando ejecutamos la
detección agresiva:
    # nmap -A <IPs>
    # nmap -O -sV -sC --traceroute
si queremos tener un informe
    # nmap -Pn -sS -A -oN fichero.txt <IPs>
    # nmap -Pn -sS -A -oA fichero.xml <IPs>
y luego convertimos el fichero en HTML
    # xsltproc fichero.xml -o resultado.html
